# XML and XPath Requirements for srcMove Output

This document defines the structural and XPath requirements for all XML files generated by this project. The goal is to ensure that the output can be reliably queried using namespace-aware XPath by downstream analysis tools.

---

# 1. Core Principle

All generated XML must be **valid srcML-compatible XML** that preserves **srcDiff structure** and adds **srcMove metadata** without breaking existing XPath queries.

In other words:

srcMove = srcDiff + move annotations

The document must remain a valid srcDiff-style document while providing additional attributes in the srcMove namespace.

---

# 2. Required Namespaces

Every output XML file must define the following namespaces on the root element.

```xml
xmlns="http://www.srcML.org/srcML/src"
xmlns:diff="http://www.srcML.org/srcDiff"
xmlns:mv="http://www.srcML.org/srcMove"
````

Optional language namespaces may appear (example):

```xml
xmlns:cpp="http://www.srcML.org/srcML/cpp"
```

Namespace meanings:

| Prefix        | Purpose                  |
| ------------- | ------------------------ |
| src (default) | srcML program structure  |
| diff          | srcDiff change markers   |
| mv            | srcMove move annotations |

Important: XPath queries must reference these namespaces explicitly.

Example:

```
/src:unit/src:unit/src:function
```

---

# 3. Root Structure

All documents must follow the srcML archive structure.

Example:

```xml
<src:unit>
    <src:unit filename="file1.cpp">
        ...
    </src:unit>

    <src:unit filename="file2.cpp">
        ...
    </src:unit>
</src:unit>
```

Requirements:

* Root element must be `unit` in the srcML namespace.
* Each file must be represented as a nested `unit` element.
* File units must include a `filename` attribute.

---

# 4. srcDiff Compatibility

All srcDiff constructs must remain intact.

Allowed elements include:

```
diff:insert
diff:delete
diff:replace
diff:ws
```

These elements wrap modified source code regions.

Example:

```xml
<diff:delete>
    <src:function> ... </src:function>
</diff:delete>
```

srcMove must **never alter the structure of srcDiff nodes**.

---

# 5. srcMove Metadata

Move detection is expressed using attributes in the `mv` namespace.

Example:

```xml
<diff:delete move="1"
             mv:partner="/src:unit/src:unit[@filename='main.cpp']/diff:insert[1]">
```

Minimum requirements:

| Attribute  | Meaning                                  |
| ---------- | ---------------------------------------- |
| move="1"   | node participates in a move              |
| mv:partner | XPath pointing to the corresponding node |

The XPath must identify the **matching move partner node**.

---

# 6. XPath Requirements

All XPath expressions stored in attributes must satisfy the following rules.

## 6.1 Namespace-aware

All element names must include prefixes.

Correct:

```
/src:unit/src:unit[@filename='foo.cpp']/diff:insert
```

Incorrect:

```
/unit/unit/diff:insert
```

---

## 6.2 Stable (avoid positional-only paths)

Paths must prefer semantic identifiers instead of numeric indices when possible.

Preferred:

```
/src:unit/src:unit[@filename="foo.cpp"]/src:function[src:name="foo"]/diff:delete
```

Avoid:

```
/src:unit/src:unit[2]/diff:delete[1]
```

Indices may be used only when no semantic identifier exists.

---

## 6.3 File identification

File nodes must always be referenced using the `filename` attribute.

Example:

```
/src:unit/src:unit[@filename="main.cpp"]
```

---

## 6.4 Function identification

When referencing functions, include the function name predicate.

Example:

```
/src:unit/src:unit[@filename="foo.cpp"]/src:function[src:name="foo"]
```

---

# 7. Expected Query Patterns

Consumers of srcMove XML should be able to run queries like:

### All moved deletes

```
 //diff:delete[@move]
```

### All moved inserts in a specific file

```
/src:unit/src:unit[@filename="main.cpp"]//diff:insert[@move]
```

### Functions involved in moves

```
 //diff:insert[@move]/src:function
```

### Retrieve move partner path

```
 //@mv:partner
```

---

# 8. Design Goals

The XML must satisfy these goals:

* valid srcML document
* valid srcDiff document
* extended with srcMove metadata
* stable XPath addressing
* easily queryable using namespace-aware XPath
* resistant to structural changes in unrelated parts of the file

---

# 9. Summary

1. All output XML must remain **valid srcML + srcDiff documents**.  
   srcMove only **adds metadata**; it must not change the srcDiff structure.

2. The root element must be a **srcML archive unit**:
   `/src:unit`.

3. Each source file must be represented as:
   `/src:unit/src:unit[@filename="..."]`.

4. Required namespaces on the root element:
   - srcML (default): `http://www.srcML.org/srcML/src`
   - diff: `http://www.srcML.org/srcDiff`
   - mv: `http://www.srcML.org/srcMove`

5. srcMove metadata must only appear as **attributes in the `mv` namespace**.

6. Move nodes must include:
   - `move="1"`
   - an XPath attribute pointing to the partner node.

7. Stored XPath expressions **must be namespace-aware**:
   use `src:` and `diff:` prefixes.

8. XPath paths must be **stable and semantic** whenever possible:
   prefer `@filename` and `src:name` predicates over numeric indices.

9. XPath expressions must identify nodes in the **actual emitted tree structure**.

10. The resulting XML must allow reliable XPath queries such as:
```xpath
//diff:delete[@move]
/src:unit/src:unit[@filename="foo.cpp"]//diff:insert[@move]
